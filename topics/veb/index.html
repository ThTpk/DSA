van Emde Boas Tree (vEB)

แนวคิด

vEB เก็บเซตของจำนวนเต็มในช่วง 0 … u−1 และทำ insert / member / successor / min / max ได้เร็วถึง O(log log u) (เร็วกว่า BST!) ด้วยการ แบ่งจักรวาลเป็น √u คลัสเตอร์ แบบเรียกซ้ำ

  • แตกเลข x เป็น high(x) = ⌊x/√u⌋ (คลัสเตอร์ไหน) และ low(x) = x mod √u (ตำแหน่งในคลัสเตอร์)
  • summary = vEB ย่อยที่จำว่าคลัสเตอร์ไหน "ไม่ว่าง" → กระโดดหาคลัสเตอร์ถัดไปได้เร็ว
  • เคล็ดลับสำคัญ: เก็บ min แยกไว้ (ไม่ใส่ลงคลัสเตอร์) → insert ลงเซตว่างเป็น O(1) ทำให้ลดการเรียกซ้ำ

💡 ตัวอย่างนี้ u=16 → √u=4 (แสดงการแบ่งระดับบนสุด: summary 4 ช่อง + 4 คลัสเตอร์ ๆ ละ 4)
