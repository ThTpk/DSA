Hash Table (ตารางแฮช)

แนวคิด

Hash Table เก็บข้อมูลโดยใช้ hash function คำนวณตำแหน่ง (bucket) จาก key ทำให้เข้าถึงได้เร็วเฉลี่ย O(1) เมื่อ key สองตัวได้ตำแหน่งเดียวกัน เรียกว่า collision วิธีแก้ที่ใช้ในตัวอย่างนี้คือ chaining (ต่อกันเป็น linked list ในแต่ละ bucket)

ความซับซ้อน: insert / search / delete เฉลี่ย O(1), แย่สุด O(n) เมื่อ collision เยอะ · ใช้ทำ dictionary, set, cache · ในตัวอย่าง hash(key) = key % 7
