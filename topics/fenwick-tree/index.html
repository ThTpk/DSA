Fenwick Tree / Binary Indexed Tree (BIT)

แนวคิด

Fenwick Tree (BIT) ตอบ ผลรวมสะสม (prefix sum) และแก้ค่าได้เร็ว O(log n) โดยใช้แค่อาเรย์เดียว (กระชับกว่า Segment Tree) เคล็ดลับคือ lowbit = i & (−i) (บิต 1 ขวาสุด) ที่บอกว่าแต่ละช่อง tree[i] ดูแลผลรวมของช่วงยาวเท่าไหร่

  • prefix(i): ไล่ i -= lowbit(i) บวกไปเรื่อยๆ จนถึง 0
  • update(i, Δ): ไล่ i += lowbit(i) บวก Δ ทุกช่องจน > n

ความซับซ้อน: query / update = O(log n) · พื้นที่ O(n) (1-indexed)

ช่อง BIT ที่กำลังเยี่ยม (ตามการกระโดด)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ระบบจัดอันดับเกม มีคะแนนผู้เล่นที่เปลี่ยนตลอด ต้องตอบ "มีผู้เล่นกี่คนที่คะแนน ≤ X" และ "บวกคะแนนให้ผู้เล่น i" ได้เร็ว

แก้ด้วย Fenwick Tree (BIT) อย่างไร: เก็บความถี่สะสม · prefix(i) ไล่ i −= i & (−i) บวกผลรวมจนถึง 0 ได้ O(log n) · point update i += i & (−i) O(log n) · ใช้แค่อาเรย์เดียว โค้ดสั้นและกินพื้นที่น้อยกว่า Segment Tree เมื่อต้องการแค่ prefix sum
