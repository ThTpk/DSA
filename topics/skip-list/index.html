Skip List (รายการข้ามชั้น)

แนวคิด

Skip List คือ linked list ที่เรียงแล้ว แต่มี "เลนด่วน" หลายชั้น ซ้อนอยู่ข้างบน แต่ละโหนดสุ่มความสูง ทำให้ชั้นบนข้ามหลายโหนดทีเดียว — ค้นหาเหมือน "ลงบันได": เดินขวาในชั้นบนจนเกือบเกิน แล้วลงชั้นล่าง ทำซ้ำจนถึงชั้นล่างสุด

ความซับซ้อน: search/insert/delete เฉลี่ย O(log n) · เป็นทางเลือกที่ง่ายกว่า balanced tree (ใช้ใน Redis, LevelDB)

ตำแหน่งปัจจุบัน เส้นทางที่เดิน เจอเป้าหมาย

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ทำ leaderboard ของเกมที่ต้องเพิ่มคะแนน, ลบผู้เล่น และดึงอันดับช่วงคะแนนได้เร็ว โดยไม่อยากเขียน balanced tree ที่ซับซ้อน

แก้ด้วย Skip List อย่างไร: เก็บคะแนนเรียงไว้ใน linked list แล้วมี "เลนด่วน" หลายชั้นสุ่มความสูง → ค้นหา/แทรก/ลบ เฉลี่ย O(log n) เหมือน "ลงบันได" ข้ามทีละหลายโหนดในชั้นบน; โค้ดง่ายกว่า red-black tree และรองรับ range query โดยตรง

เจอได้ที่ไหน: Redis sorted set (ZADD/ZRANGE), index ใน LevelDB/RocksDB, ดัชนีของ Apache Lucene · LeetCode: Design Skiplist (1206)
