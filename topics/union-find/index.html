Union-Find (Disjoint Set)

แนวคิด

Union-Find จัดการกลุ่ม (set) ที่ไม่ทับซ้อนกัน เก็บด้วยอาเรย์ parent[] แต่ละสมาชิกชี้ไปยัง "พ่อ" จนถึงตัวแทนกลุ่ม (root) ที่ชี้ตัวเอง มี 2 การทำงานหลัก:

  • find(x): ตามตัวชี้ขึ้นไปจน root → บอกว่า x อยู่กลุ่มไหน
  • union(a,b): รวมกลุ่มของ a และ b เข้าด้วยกัน (ให้ root หนึ่งชี้อีก root)

Path compression ทำให้ตัวบนเส้นทางชี้ตรงไป root เลย เร็วขึ้นมาก · ความซับซ้อนเฉลี่ยเกือบ O(1) (α(n)) · ใช้ใน Kruskal MST, ตรวจ connected components

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: มี n คน ให้ความสัมพันธ์ว่าใครเป็นเพื่อนใคร (เพื่อนของเพื่อนถือเป็นกลุ่มเดียวกัน) ถามว่ามีกลุ่มเพื่อน (friend circles) ทั้งหมดกี่กลุ่ม

แก้ด้วย Union-Find อย่างไร: เริ่มให้ทุกคนเป็นกลุ่มของตัวเอง (n กลุ่ม) ทุกคู่ที่เป็นเพื่อนกันก็ union() เข้าด้วยกัน จำนวนกลุ่มสุดท้าย = จำนวน root ที่ต่างกัน · ด้วย path compression + union by rank ทุก operation เกือบ O(1) (α(n)) · ตรวจว่าสองโหนดอยู่กลุ่มเดียวกันไหมได้ทันที
