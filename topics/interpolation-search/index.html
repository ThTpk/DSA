Interpolation Search (ค้นหาแบบประมาณตำแหน่ง)

แนวคิด

Interpolation Search คล้าย Binary Search แต่ฉลาดกว่า: แทนที่จะดูตรงกลางเสมอ มัน ประมาณตำแหน่ง ที่เป้าหมายน่าจะอยู่จากค่า โดยใช้สูตรเชิงเส้น pos = lo + (target − A[lo])·(hi − lo) / (A[hi] − A[lo])

ความซับซ้อน: เฉลี่ย O(log log n) เมื่อข้อมูล กระจายสม่ำเสมอ, แย่สุด O(n) เมื่อข้อมูลเอียงมาก · ต้องเป็นอาเรย์ที่เรียงแล้ว

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ในสมุดโทรศัพท์เรียงเลขที่กระจายสม่ำเสมอ [10,20,30,...,1000] หาเลข 700

แก้ด้วย Interpolation Search อย่างไร: ประมาณตำแหน่งจากสัดส่วนค่าจริง (เหมือนเปิดสมุดโทรศัพท์ไปใกล้ตัวที่ค้นทันที) แทนการดูตรงกลางเสมอ · O(log log n) เมื่อข้อมูลกระจายสม่ำเสมอ, แย่สุด O(n) · เหมาะกับข้อมูลเรียงและกระจายเชิงเส้น

เจอได้ที่ไหน: ค้นใน index ตัวเลขที่กระจายสม่ำเสมอ · ฐานข้อมูลคีย์เรียงต่อเนื่อง · ดีกว่า binary search เมื่อข้อมูล uniform
