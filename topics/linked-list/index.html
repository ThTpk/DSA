Linked List (รายการโยง)

แนวคิด

Linked List เก็บข้อมูลเป็น โหนด (node) ที่กระจายในหน่วยความจำ แต่ละโหนดเก็บค่าและ ตัวชี้ (pointer) ไปยังโหนดถัดไป โหนดสุดท้ายชี้ไป null การแทรก/ลบทำได้เร็วถ้ารู้ตำแหน่ง (แค่เปลี่ยน pointer) แต่การเข้าถึงต้องไล่จาก head

ความซับซ้อน: เข้าถึง/ค้นหา = O(n) (ไล่ทีละโหนด) · แทรก/ลบที่หัว = O(1) · แทรก/ลบที่ตำแหน่งใดๆ = O(n) (ต้องไล่ไปก่อน)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ทำเพลย์ลิสต์เพลงที่ผู้ใช้แทรกเพลงใหม่ตรงกลางบ่อย ๆ โดยไม่อยากเลื่อนข้อมูลทั้งก้อน

แก้ด้วย Linked List อย่างไร: เมื่อรู้โหนดก่อนหน้าแล้ว แทรก/ลบทำได้แค่เปลี่ยน pointer → O(1) ไม่ต้องเลื่อนสมาชิกเหมือน array; แต่การไปถึงโหนดที่ต้องการต้องไล่จาก head → O(n) · เหมาะเมื่อแทรก/ลบบ่อยและไม่ต้องเข้าถึงด้วย index

เจอได้ที่ไหน: โครงสร้างพื้นฐานของ stack/queue, การจัดการบล็อกหน่วยความจำ, ประวัติการเล่นเพลง · LeetCode: Reverse Linked List (206), Merge Two Sorted Lists (21), Remove Nth Node From End (19)
