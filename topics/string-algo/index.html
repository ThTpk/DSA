String Matching — KMP

แนวคิด

การค้นหา แพตเทิร์น ในข้อความ แบบตรงไปตรงมา (naive) เมื่อเจอตัวไม่ตรงต้องเลื่อนทีละ 1 และเริ่มเทียบใหม่ตั้งแต่ต้น → O(n·m)

KMP ฉลาดกว่า: เตรียมตาราง LPS (Longest Proper Prefix ที่เป็น Suffix ด้วย) ของแพตเทิร์นไว้ก่อน เมื่อเจอตัวไม่ตรง จะ ข้ามไปเทียบต่อ โดยไม่ย้อนกลับไปอ่านข้อความซ้ำ → O(n + m)

💡 แถวบน = ข้อความ · แถวล่าง = แพตเทิร์น (เลื่อนตามการเทียบ) · เขียว = ตรง · แดง = ไม่ตรง

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: หาตำแหน่งแรกที่แพตเทิร์น "abcab" ปรากฏในข้อความ "ababcabcab" (string matching)

แก้ด้วย KMP อย่างไร: สร้างตาราง prefix-function (failure function) ของแพตเทิร์นก่อน O(m) เมื่อเทียบไม่ตรงก็เลื่อนแพตเทิร์นโดยไม่ถอย pointer ของข้อความ → รวม O(n+m) เร็วกว่า brute-force O(n·m) เพราะใช้ข้อมูลที่เทียบไปแล้วซ้ำ

เจอได้ที่ไหน: ฟังก์ชัน find/indexOf ในภาษาโปรแกรม · ตรวจ DNA sequence · LeetCode 28 (Find the Index of the First Occurrence) · LeetCode 459 (Repeated Substring Pattern)
