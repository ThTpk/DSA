Red-Black Tree

แนวคิด

Red-Black Tree เป็น BST ที่ปรับสมดุลเองอีกแบบ (ใช้จริงใน std::map, Java TreeMap) ทุกโหนดมีสี แดง หรือ ดำ ตามกติกา:

  • รากเป็นสีดำเสมอ · โหนดแดงห้ามมีลูกแดง (ห้ามแดงติดแดง)
  • ทุกเส้นทางจากโหนดลงไปถึงใบ ต้องผ่านโหนดดำจำนวนเท่ากัน (black-height)

เมื่อ insert แล้วผิดกติกา จะซ่อมด้วยการ ทาสีใหม่ (recolor) และ หมุน (rotation) · รับประกัน O(log n) · ปรับสมดุลน้อยครั้งกว่า AVL จึงนิยมในงานที่ insert/delete บ่อย
