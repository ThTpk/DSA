Red-Black Tree

แนวคิด

Red-Black Tree เป็น BST ที่ปรับสมดุลเองอีกแบบ (ใช้จริงใน std::map, Java TreeMap) ทุกโหนดมีสี แดง หรือ ดำ ตามกติกา:

  • รากเป็นสีดำเสมอ · โหนดแดงห้ามมีลูกแดง (ห้ามแดงติดแดง)
  • ทุกเส้นทางจากโหนดลงไปถึงใบ ต้องผ่านโหนดดำจำนวนเท่ากัน (black-height)

เมื่อ insert แล้วผิดกติกา จะซ่อมด้วยการ ทาสีใหม่ (recolor) และ หมุน (rotation) · รับประกัน O(log n) · ปรับสมดุลน้อยครั้งกว่า AVL จึงนิยมในงานที่ insert/delete บ่อย

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ต้องการ map ที่ key เรียงตลอดเวลา รองรับ insert/delete/lookup บ่อยมาก และยังต้องวนค่าตามลำดับได้ — จะเลือกโครงสร้างไหน

แก้ด้วย Red-Black Tree อย่างไร: เป็น balanced BST ที่ซ่อมสมดุลด้วย recolor + rotation รับประกัน O(log n) ทุกการดำเนินการ · ปรับสมดุลน้อยครั้งกว่า AVL (หมุนไม่เกิน 2–3 ครั้งต่อ insert) จึงเหมาะกับงาน insert/delete หนัก · inorder ให้ค่าตามลำดับ

เจอได้ที่ไหน: C++ std::map/std::set, Java TreeMap/TreeSet, Linux CFS scheduler, จัดการช่วงหน่วยความจำในเคอร์เนล (epoll, vma)
