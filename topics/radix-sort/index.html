Radix Sort (การเรียงตามหลัก)

แนวคิด

Radix Sort เรียงจำนวนเต็มทีละหลัก จาก หลักหน่วย → สิบ → ร้อย (LSD) แต่ละรอบใช้การแจกเข้า ถัง 0–9 ตามเลขหลักนั้นแบบ เสถียร (stable) แล้วเก็บกลับตามลำดับถัง พอครบทุกหลักก็เรียงเสร็จ

ความซับซ้อน: O(d·(n + b)) เมื่อ d = จำนวนหลัก, b = ฐาน (10) · เร็วมากเมื่อเลขไม่ยาว · ใช้ Counting Sort เป็นตัวช่วยในแต่ละหลัก

หลักที่กำลังดู (sub = เลขหลักนั้น)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: เรียงรหัสไปรษณีย์/จำนวนเต็ม [170, 45, 75, 90, 2, 802, 24, 66] จากน้อยไปมากด้วย Radix Sort

แก้ด้วย Radix Sort อย่างไร: เรียงทีละหลักจากหลักหน่วย (LSD) ด้วย Counting Sort แบบ stable · หลักหน่วย: [170,90,2,802,24,45,75,66] · หลักสิบ: [2,802,24,45,66,170,75,90] · หลักร้อย: [2,24,45,66,75,90,170,802] · เสร็จ · เพราะแต่ละหลักเรียงแบบ stable ลำดับจากหลักก่อนหน้าจึงไม่เสีย · O(d·(n + b)) เมื่อ d=จำนวนหลัก, b=ฐาน(10) · เร็วกว่าการเรียงแบบเปรียบเทียบเมื่อตัวเลขไม่ยาว

เจอได้ที่ไหน: เรียงจำนวนเต็ม/สตริงความยาวคงที่จำนวนมาก (รหัสไปรษณีย์, เลขบัตร, IP), DBMS ที่เรียง integer key, เรียง suffix array · LeetCode: Maximum Gap, Sort an Array (เมื่อค่าเป็น integer ช่วงกว้างแต่หลักน้อย)
