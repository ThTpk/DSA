LRU Cache (แคชแบบใช้ล่าสุด)

แนวคิด

LRU Cache เก็บข้อมูลได้จำกัด เมื่อเต็มจะ ไล่ตัวที่ไม่ได้ใช้นานสุด (Least Recently Used) ออก ทำได้เร็ว O(1) ด้วยการรวม HashMap (หาเร็ว) + Doubly Linked List (เรียงตามความใหม่)

  • get / put ตัวที่ถูกใช้จะถูกย้ายไปไว้ หน้าสุด (MRU)
  • เมื่อใส่เกินความจุ → ลบ ตัวท้ายสุด (LRU) ทิ้ง

ใช้ใน: แคชของ OS, เบราว์เซอร์, ฐานข้อมูล · ความจุในตัวอย่างนี้ = 4

ซ้ายสุด = MRU (ใช้ล่าสุด) · ขวาสุด = LRU (จะถูกไล่ก่อน)
