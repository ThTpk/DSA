Ternary Search (ค้นหาแบบสามส่วน)

แนวคิด

Ternary Search คล้าย Binary Search แต่แบ่งช่วงค้นหาออกเป็น 3 ส่วน ด้วยจุดตัด 2 จุด คือ mid1 = lo + (hi−lo)/3 และ mid2 = hi − (hi−lo)/3 จากนั้นเทียบเป้าหมายกับค่าที่ทั้งสองจุด เพื่อ ตัดทิ้งหนึ่งในสามส่วน ในแต่ละรอบ

ความซับซ้อน: O(log₃ n) รอบ แต่เทียบ 2 ครั้งต่อรอบ จึง ช้ากว่า Binary Search เล็กน้อย ในทางปฏิบัติ · แนวคิด "แบ่งสามส่วน" นี้ใช้บ่อยกับการหา จุดสูงสุด/ต่ำสุดของฟังก์ชัน unimodal

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: หาจุดสูงสุดของฟังก์ชันรูประฆัง (unimodal) เช่น f(x) = -(x-3)² + 5 บนช่วง [0,6]

แก้ด้วย Ternary Search อย่างไร: แบ่งช่วงเป็นสามส่วนด้วยจุด m1, m2 ถ้า f(m1) < f(m2) ตัดส่วนซ้ายทิ้ง มิฉะนั้นตัดขวา · O(log n) · ใช้ได้เพราะฟังก์ชันมียอดเดียว (unimodal) ขึ้นแล้วลง
