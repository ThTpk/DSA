Floyd-Warshall (เส้นทางสั้นสุดทุกคู่)

แนวคิด

Floyd-Warshall หาเส้นทางสั้นสุด ระหว่างทุกคู่โหนด พร้อมกัน ด้วย DP บนตารางระยะทาง: ลองให้แต่ละโหนด k เป็น "จุดพักกลางทาง" แล้วถามว่า "ไป i→k→j สั้นกว่า i→j เดิมไหม?"

dist[i][j] = min( dist[i][j], dist[i][k] + dist[k][j] ) ทำสำหรับทุก k, i, j

ความซับซ้อน: O(V³) · เข้าใจง่าย เขียนสั้น · เหมาะกับกราฟเล็ก-กลางที่ต้องการระยะทุกคู่ · รองรับน้ำหนักลบ (แต่ห้ามมี negative cycle)

ช่อง dist[i][j] ที่กำลังคิด ช่องที่ใช้ (ผ่าน k)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: มีเมือง n เมือง (n เล็ก ~100) และระยะถนนระหว่างกัน ต้องตอบคำถามซ้ำๆ ว่า "จากเมือง i ไป j สั้นสุดเท่าไร" สำหรับทุกคู่ที่ถามมา

แก้ด้วย Floyd-Warshall อย่างไร: คำนวณระยะสั้นสุดทุกคู่ครั้งเดียวด้วย DP: ลองให้แต่ละโหนด k เป็นจุดพักกลางทาง dist[i][j]=min(dist[i][j], dist[i][k]+dist[k][j]) วนทุก k,i,j · O(V³) เขียนสั้นมาก · หลังคำนวณตอบทุก query ได้ O(1) · รองรับน้ำหนักลบ (ห้ามมี negative cycle)
