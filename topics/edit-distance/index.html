Edit Distance (Levenshtein)

แนวคิด

Edit Distance = จำนวนการแก้ไขน้อยที่สุดในการเปลี่ยนสตริงหนึ่งเป็นอีกสตริง โดยใช้ 3 การกระทำ: แทรก (insert), ลบ (delete), แทนที่ (replace)

dp[i][j] = ระยะแก้ไขระหว่าง s1[0..i] กับ s2[0..j] · กฎ:

  • ตัวอักษรตรงกัน → dp[i-1][j-1] (ไม่ต้องแก้)
  • ไม่ตรง → 1 + min(แทนที่ dp[i-1][j-1], ลบ dp[i-1][j], แทรก dp[i][j-1])

ใช้ใน: ตรวจคำสะกด, autocorrect, diff, ชีวสารสนเทศ (DNA) · O(m·n)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: แปลงคำว่า "kitten" เป็น "sitting" ต้องแก้ (แทรก/ลบ/แทนที่ตัวอักษร) น้อยที่สุดกี่ครั้ง (คำตอบ: 3)

แก้ด้วย Edit Distance (Levenshtein) อย่างไร: dp[i][j] = ระยะแก้ของ s1[..i] กับ s2[..j] · ถ้าตัวอักษรเท่ากันใช้ค่าทแยงเดิม ไม่เท่าใช้ 1 + min(แทรก, ลบ, แทนที่) · O(m·n)
