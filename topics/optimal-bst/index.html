หมวด 6 · เทคนิคออกแบบอัลกอริทึม

Optimal Binary Search Tree (DP)

แนวคิด

ถ้ารู้ว่าแต่ละคีย์ถูกค้นหา บ่อยแค่ไหน (frequency) เราอยากสร้าง BST ที่ทำให้ ต้นทุนค้นหารวมน้อยสุด (คีย์ที่ค้นบ่อยควรอยู่ตื้น) ใช้ DP: cost[i][j] = W(i,j) + min over r ( cost[i][r−1] + cost[r+1][j] ) โดย r = รากของซับทรี และ W(i,j) = ผลรวม frequency ของคีย์ i..j

ความซับซ้อน: O(n³) · ต้นทุน = ผลรวมของ (frequency × ความลึก) ของทุกคีย์
