Huffman Coding (การบีบอัดแบบฮัฟฟ์แมน)

แนวคิด

Huffman Coding บีบอัดข้อมูลโดยให้ตัวอักษรที่ พบบ่อย ใช้รหัสบิตสั้น ตัวที่พบน้อยใช้รหัสยาว (greedy) สร้างต้นไม้โดย รวม 2 โหนดที่ความถี่น้อยที่สุด เข้าด้วยกันซ้ำๆ (ใช้ min-heap) จนเหลือต้นเดียว แล้วเดินซ้าย=0 ขวา=1 ได้รหัสของแต่ละตัว

ความซับซ้อน: O(n log n) · เป็น prefix code (ไม่มีรหัสไหนเป็นคำนำหน้าของอีกรหัส) · ใช้ใน ZIP, JPEG, MP3

2 โหนดที่กำลังรวม โหนดใหม่จากการรวม

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ข้อความ "AAAABBBCCD" (A=4, B=3, C=2, D=1) เข้ารหัสอย่างไรให้จำนวนบิตรวมน้อยที่สุด

แก้ด้วย Huffman Coding อย่างไร: ใช้ greedy + min-heap รวมสองโหนดความถี่น้อยสุดเป็นต้นไม้ซ้ำๆ · ตัวที่พบบ่อยได้รหัสสั้น (prefix-free code) · O(n log n) · ให้รหัสที่สั้นที่สุดเชิงทฤษฎี (optimal)

เจอได้ที่ไหน: บีบอัดไฟล์ ZIP/GZIP · JPEG/MP3 · DEFLATE · เป็นตัวอย่างคลาสสิกของ greedy + priority queue
