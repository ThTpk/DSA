Johnson's Algorithm (เส้นทางสั้นสุดทุกคู่)

แนวคิด

หาเส้นทางสั้นสุด ทุกคู่โหนด บนกราฟมีทิศ ที่มีน้ำหนักลบได้ · เร็วกว่า Floyd-Warshall บนกราฟเบาบาง (O(V²log V + VE) เทียบ O(V³)) ด้วยกลเม็ด reweighting:

  • เพิ่มโหนดเสมือน q ที่ชี้ไปทุกโหนดด้วยน้ำหนัก 0 แล้วรัน Bellman-Ford หา h(v)
  • ปรับน้ำหนัก w'(u,v) = w(u,v) + h(u) − h(v) → ไม่มีลบ (รัน Dijkstra ได้)
  • รัน Dijkstra จากทุกโหนด แล้วแปลงกลับ d(u,v) = d'(u,v) − h(u) + h(v)

💡 ตัวเลขใต้โหนด = h(v) · ป้ายบนเส้น = น้ำหนักที่ปรับแล้ว · ตารางขวา = ระยะทุกคู่

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: กราฟมีทิศขนาดใหญ่แต่เบาบาง (V=2000, E≈8000) มีน้ำหนักลบบางเส้น (เช่น โปรโมชันลดราคา) ต้องการระยะสั้นสุดทุกคู่โหนด

แก้ด้วย Johnson's อย่างไร: Floyd-Warshall จะเป็น O(V³)=8×10⁹ ช้าไป · Johnson's ใช้ reweighting: เพิ่มโหนดเสมือน q ชี้ทุกโหนดน้ำหนัก 0 รัน Bellman-Ford หา h(v) ปรับเป็น w'=w+h(u)−h(v) (ไม่มีลบ) แล้วรัน Dijkstra จากทุกโหนด · O(V²log V + VE) เร็วกว่ามากบนกราฟ sparse · รองรับน้ำหนักลบ (ห้ามมี negative cycle)

เจอได้ที่ไหน: all-pairs shortest path บนกราฟ sparse ที่มีน้ำหนักลบ · ตารางระยะทาง/ค่าโดยสารทั้งเครือข่าย · เปรียบเทียบกับ Floyd-Warshall (เหมาะกราฟแน่น/เล็ก)
