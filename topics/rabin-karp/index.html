Rabin-Karp (ค้นหาด้วย rolling hash)

แนวคิด

Rabin-Karp เทียบ ค่าแฮช (hash) ของแพตเทิร์นกับหน้าต่างในข้อความแทนการเทียบตัวอักษรทุกตัว ใช้ rolling hash เลื่อนหน้าต่างทีละ 1 แล้วอัปเดตแฮชเร็วๆ (ลบตัวที่ออก บวกตัวที่เข้า) ถ้าแฮชตรงค่อย ตรวจตัวอักษรยืนยัน (กัน hash ชนกันโดยบังเอิญ)

ความซับซ้อน: เฉลี่ย O(n + m), แย่สุด O(n·m) (เมื่อชนกันบ่อย) · ในตัวอย่างนี้ใช้ hash = ผลรวมรหัสตัวอักษร · เด่นเรื่องค้นหาหลายแพตเทิร์นพร้อมกัน

หน้าต่างปัจจุบัน ตรง ไม่ตรง

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ตรวจว่าเอกสาร 2 ฉบับมีย่อหน้าซ้ำกันไหม โดยเทียบสตริงยาว ๆ จำนวนมาก

แก้ด้วย Rabin-Karp อย่างไร: แปลงแต่ละ substring เป็นค่าแฮชด้วย rolling hash เลื่อนหน้าต่างทีละตัวคำนวณแฮชใหม่ใน O(1) → เฉลี่ย O(n+m) เทียบแฮชก่อน ตรงค่อยเทียบตัวอักษรจริงกันแฮชชนกัน เหมาะเมื่อต้องหาหลายแพตเทิร์นพร้อมกัน
