Jump Search (ค้นหาแบบกระโดด)

แนวคิด

Jump Search ใช้กับอาเรย์ที่ เรียงแล้ว โดยกระโดดทีละบล็อกขนาด ⌊√n⌋ จนเจอบล็อกที่ค่าปลายบล็อก ≥ เป้าหมาย แล้วค่อย ค้นเชิงเส้น ภายในบล็อกนั้น เร็วกว่า Linear Search แต่ช้ากว่า Binary Search

ความซับซ้อน: O(√n) · เหมาะเมื่อการ "กระโดดถอยหลัง" แพง (เช่นบางระบบจัดเก็บ)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: ในอาเรย์เรียงแล้ว 10,000 ตัว หาเลข 4321 โดยกระโดดทีละช่วงแทนการดูทุกตัว

แก้ด้วย Jump Search อย่างไร: กระโดดทีละ √n (ที่นี่ 100) จนเลยค่าเป้าหมาย แล้วถอยมาค้นเชิงเส้นในบล็อกนั้น · O(√n) · เหมาะกับข้อมูลเรียงแล้วที่ "ย้อนกลับแพง" (เช่นเทปสื่อบันทึก) ซึ่งกระโดดไปข้างหน้าถูกกว่า binary search

เจอได้ที่ไหน: ค้นในข้อมูลเรียงแล้วบนสื่อที่ seek แพง · ระบบที่ binary search กระโดดไกลเกินไม่คุ้ม · เป็นจุดกึ่งกลางระหว่าง linear กับ binary search
