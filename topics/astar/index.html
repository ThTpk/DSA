A* Search (ค้นหาเส้นทางด้วย heuristic)

แนวคิด

A* (A-star) คือ Dijkstra ที่ฉลาดขึ้น ด้วยการเดา "ระยะที่เหลือ" ไปยังเป้าหมาย แต่ละช่องเก็บ 3 ค่า: g = ระยะจริงจากจุดเริ่ม, h = ค่าประมาณระยะที่เหลือ (heuristic, ที่นี่ใช้ระยะแมนฮัตตัน), และ f = g + h ทุกรอบ A* จะเลือกขยายช่องที่มี f น้อยที่สุด

ความซับซ้อน: ขึ้นกับ heuristic — ถ้า h ไม่ประเมินเกินจริง (admissible) A* รับประกันเส้นทางสั้นสุด · เทียบ: ถ้า h = 0 ทุกช่อง A* จะกลายเป็น Dijkstra พอดี

กำลังขยาย (f ต่ำสุด) open set (รอพิจารณา) closed set (เสร็จแล้ว) เส้นทางสั้นสุด กำแพง
g ระยะจากจุดเริ่ม h ระยะประมาณที่เหลือ f = g + h (เลขใหญ่กลางช่อง)

📝 ตัวอย่างโจทย์ & การนำไปใช้

โจทย์: หาเส้นทางสั้นสุดของตัวละครในเกมจากจุด A ไปจุด B บนกริดที่มีกำแพงขวาง

แก้ด้วย A* อย่างไร: เลือกขยายโหนดที่ f(n) = g(n) + h(n) ต่ำสุด โดย g คือระยะที่เดินมาแล้ว h คือฮิวริสติกประมาณระยะเหลือ (เช่น Manhattan distance) · เร็วกว่า Dijkstra เพราะ h ชี้นำให้ค้นไปทางเป้าหมาย · ได้คำตอบดีที่สุดเมื่อ h ไม่ประเมินเกินจริง (admissible)
