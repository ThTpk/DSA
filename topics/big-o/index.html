Big-O Notation (อัตราการเติบโต)

แนวคิด

Big-O อธิบายว่าเวลา/หน่วยความจำที่อัลกอริทึมใช้ เติบโตเร็วแค่ไหน เมื่อขนาดข้อมูล n ใหญ่ขึ้น โดยมองที่แนวโน้มหลัก (ตัด constant และพจน์เล็กทิ้ง) ยิ่งโตช้า ยิ่งดี: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)

💡 เลื่อน n ดูว่าจำนวนการทำงานต่างกันมหาศาลแค่ไหน (แกนตั้งเป็น log scale)
