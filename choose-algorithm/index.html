🧭 วิธีเลือกอัลกอริทึม

อ่านโจทย์แล้วจับสัญญาณว่าต้องใช้อัลกอริทึมตระกูลไหน — ทักษะที่สำคัญที่สุดของการแข่ง/สัมภาษณ์ สำคัญกว่าการเขียนโค้ดเสียอีก

🪜 กรอบคิด 4 ขั้น (ใช้ได้กับทุกโจทย์)

1
โจทย์ถามอะไร?
หาค่าน้อย/มากสุด? · นับจำนวนวิธี? · เป็นไปได้ไหม/หาคำตอบสักอัน? · สร้าง/จัดเรียง?
2
ขนาดข้อมูล n เท่าไร?
บอก เป้าหมาย Big-O ที่ยอมรับได้ — สัญญาณที่ทรงพลังที่สุด (ดูตารางที่ 1)
3
ข้อมูลโครงสร้างแบบไหน?
กราฟ/เครือข่าย? · เรียงได้ไหม? · ต้นไม้? · ช่วงเวลา? · กริด? · ตัวเลข/จำนวนเฉพาะ? · จุด/รูปทรง?
4
มีคำใบ้อะไร?
keyword ในโจทย์ที่ชี้ตรง ๆ ไปยังตระกูลอัลกอริทึม (ดูตารางที่ 2)

📏 ตารางที่ 1 — ดู “ขนาด n” เดาเป้าหมาย Big-O

สมมติคอมพิวเตอร์ทำงานราว 10⁸ ครั้ง/วินาที — เห็นค่า n ก็เดาความซับซ้อนที่ต้องได้ และตัดอัลกอริทึมที่ช้าเกินทิ้งได้ทันที

n สูงสุดBig-O ที่ต้องได้อัลกอริทึมที่ใช่
≤ 11O(n!)Permutations, brute force
≤ 20–25O(2ⁿ)Backtracking, Bitmask DP
≤ 100–500O(n³)Floyd-Warshall, Matrix-Chain, interval DP
≤ 2,000–5,000O(n²)DP, Edit Distance
≤ 10⁵–10⁶O(n log n)Sort, Binary Search, Segment Tree, Dijkstra, Greedy
≤ 10⁷–10⁸O(n)Two Pointers, Sliding Window, Kadane
สูงมาก (10¹⁸)O(log n) / สูตรModular Exp, GCD, คณิตศาสตร์

💡 เห็น n ≤ 18 → แทบฟันธงเป็น bitmask/backtracking · เห็น n ≤ 10⁶ → ห้ามคิด O(n²) เด็ดขาด

🎯 ตารางที่ 2 — ดู “โจทย์ถามอะไร + คำใบ้” เดาตระกูลอัลกอริทึม

สัญญาณในโจทย์อัลกอริทึมที่น่าจะใช่
“เส้นทาง/ต้นทุนน้อยที่สุด” บนเครือข่ายDijkstra (น้ำหนักบวก) · Bellman-Ford (ติดลบ) · BFS (ไร้น้ำหนัก)
“จำนวนก้าวน้อยสุดไปถึงสถานะหนึ่ง”BFS บน state graph
“มาก/น้อยสุด” + ปัญหาย่อยซ้อนทับ + “นับจำนวนวิธี”Dynamic Programming
“เลือกแบบโลภแล้วน่าจะดีสุด” (เรียงแล้วคว้า)Greedy (ต้องพิสูจน์ greedy-choice)
“เป็นไปได้ไหม / หาคำตอบสักชุด” + ลองแล้วถอยBacktracking
“ถามค่าในช่วง [l..r] + แก้ค่า” บ่อย ๆSegment Tree · Fenwick Tree
“กลุ่ม / เชื่อมต่อกันไหม / component”Union-Find · DFS/BFS
“จุดที่ขาดแล้วกราฟแยก”Articulation / Bridges
“ค่ามากสุดที่ยังผ่านเงื่อนไข” (monotonic)Binary search on answer
“คู่/ช่วงต่อเนื่องบนข้อมูลเรียงแล้ว”Two Pointers · Sliding Window
“หา pattern ในข้อความ”KMP · Rabin-Karp
“k-th น้อยสุด / มัธยฐาน”Quickselect · Heap
“สองผู้เล่นเล่นดีที่สุด / expected value ภายใต้การเลือกดีสุด”Minimax · ทฤษฎีเกม · value iteration
จุด / เส้น / รูปหลายเหลี่ยมConvex Hull · Closest Pair · เรขาคณิต

🏆 ลองใช้จริงกับโจทย์ ICPC World Finals

ไล่ตามกรอบคิด: เห็น “สัญญาณ” → สรุป “อัลกอริทึม”

🎲 Alea Iacta Est (2023)
สัญญาณ: “ใช้จำนวนการทอยน้อยที่สุดไปให้ถึงเป้าหมาย” + สถานะลูกเต๋ามีจำกัด (7ᵈ)
→ “น้อยที่สุดไปถึงสถานะ” = shortest path · มองสถานะเป็นโหนด → Dijkstra / Bellman-Ford
🌉 Zoo Management (2022)
สัญญาณ: ข้อมูลเป็นกราฟ + ต้องดู “ส่วนที่ขาดแล้วแยก / cycle”
→ keyword bridge/cycle/component → Articulation/Bridges แล้วแก้ทีละ 2-edge-connected component
🔦 Bridging the Gap (2023)
สัญญาณ: ถาม “เวลารวมน้อยที่สุด” (optimize) + ต้องเลือกว่าใครข้ามไป-กลับ + n เล็ก
→ “เลือกแบบโลภดูจะดี แต่ต้องพิสูจน์” → Greedy ผสม DP (greedy ล้วนอาจพลาด)
⏰ Jet Lag (2023)
สัญญาณ: จัดตาราง “ช่วงเวลา” ให้ได้เงื่อนไข + n ใหญ่ (โค้ดสั้น)
→ ปัญหา interval + เลือกทีละช่วง → Greedy (มักเรียงตามเวลาจบแล้วคว้า)
💡 เคล็ดลับนักแข่ง: อ่าน constraint ก่อนอ่านโจทย์ให้จบด้วยซ้ำ — ตัวเลข n บอกครึ่งหนึ่งของคำตอบแล้ว จากนั้นค่อยจับ keyword ว่าโจทย์ “ถามอะไร” เพื่อเลือกตระกูลอัลกอริทึม
